Common Tasks

Python and R data types cannot be thought of as exact matches.

A critical difference is that Python lists are not inherently homogeneous, which is the case for R vectors.

Python R
list(zip(list1, list2)) mapply(c, vector1, vector2, SIMPLIFY = FALSE, USE.NAMES = FALSE)
dict(zip(list1, list2)) names(vector1) <- vector2
dict(zip(list1, list2)) setNames(vector1, vector2)
enumerate(list1) <generator> seq_along(list1)
max(dict.items(), lambda x = x[1]) named_vector[named_vector == max(named_vector)]
library(purrr)
list(zip(list1, list2)) map2(vector1, vector2, c)
max(dict.items(), lambda x = x[1]) named_vector %>% keep(~ .x == max(named_vector))

days <- c("Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday")

for(i in seq_along(days)) {                   # a Python-style loop works...
  print(str_glue('{i}. {days[[i]]}'))         # but it's awkward in that you would probably

}                                             # but we don't explicitly `print()` much in R

In R, functions like print() or plot() “return” side-effects rather than values.

Advanced R - Functions: Return values

With that in mind, the best solution is to use purrr::walk(), which is designed specifically to iterate over values calling a function for its side-effects.

walk(days, ~ print(str_glue("{match(.x, days)}. {.x}"))) # print() in R is a side-effect, 
## 1. Monday
## 2. Tuesday
## 3. Wednesday
## 4. Thursday
## 5. Friday
## 6. Saturday
## 7. Sunday

Underneath the hood, walk() is literally just a for() loop, but it’s consistent with a family of functions from {purrr} that provide powerful iteration tools and solutions to complicated tasks involving vectors, lists, and data.frames.

Write Lines

Python

to_write = 'Here is\nsome\nexample text\nto write'
with open('test-data/example.txt', mode='w') as file:
    file.writelines(to_write)

R

library(readr)

to_write <- 'Here is\nsome\nexample text\nto write'
write_lines(to_write, path = "test-data/example.txt")

Read Lines

Python

text = [line.strip() for line in text]     # strip whitespace indicating "new line" (\n)
print(text)
