Load Packages

  • Our trusty {tidyverse} team of packages for a ninjillion helpful functions.
  • {readxl}, as the data are stored in a .xlsx file with readxl::read_xlsx}.
  • {httr}, to programtically access the data with httr::GET().
    • I recall reading somewhere that updates were in store for {readxl} that would allow it to to read files directly from URLs, but I honestly haven’t gotten around to looking.
  • {webshot}, to be able to easily give context to the webpages mentioned in the beginning with webshot::webshot().
  • {knitr} and {kableExtra}, to prettify outputted tables with knitr::kable() and kableExtra::kablestyling()
library(tidyverse)
library(readxl)
library(httr)
library(webshot)
library(knitr)
library(kableExtra)

Get That Data

If you navigate to the referenced GitHub page where the data are kept, you’ll see the following:

webshot("https://github.com/rfordatascience/tidytuesday", 
        vheight = 1200,
        cliprect = "viewport")

Click the data link, which will take you here:

webshot("https://github.com/rfordatascience/tidytuesday/tree/master/data")

We want the file named “tidy_tuesday_week2.xlsx”. Go ahead and click that link, taking you here:

webshot("https://github.com/rfordatascience/tidytuesday/blob/master/data/tidy_tuesday_week2.xlsx")

Instead of manually downloading the file and putting it in a folder somewhere, right-click on “View Raw” and copy the link’s address. We’ll use that to access the data from within R.

Once you have that, let’s assign it to variable that I’ll call data_url.

data_url <- "https://github.com/rfordatascience/tidytuesday/blob/master/data/tidy_tuesday_week2.xlsx?raw=true"

There is likely (read: probably) a better way to do this, but I’m going to take data_url and pipe it %>% to {httr}’s GET(). I’ll then set GET()’s config = argument to write it (with httr::write_disk() to a temporary file (that I create with base::tempfile()). I’ll also give the that file a a ".xlsx" extension with the fileext = argument. I’ll then assign the result to a global variable that I’ll call temp_file with <<- (HEAVEN FORBIRD!).

That’s a lot, but here’s what it looks like:

data_url %>% 
  GET(config = write_disk(temp_file <<- tempfile(fileext = ".xlsx")))
## Response [https://raw.githubusercontent.com/rfordatascience/tidytuesday/master/data/tidy_tuesday_week2.xlsx]
##   Date: 2018-04-10 13:40
##   Status: 200
##   Content-Type: application/octet-stream
##   Size: 70.7 kB
## <ON DISK>  C:\Users\Windows\AppData\Local\Temp\RtmpyMXbdi\file1a38773e80.xlsx

We now have the .xlsx file stored in our temp_file, which we can read into R as a tibble with readxl::read_xlsx(). The column headers are, shall we say, very Excel user, so I’m going to clean those up immediately to save headaches and endless variable-referencing with a bunch of ticks (`).

Set Better Column Names

By using dplyr::rename_all(), we can rename all of our columns with a function. If we wrap them in dplyr::funs(), we can pass multiple functions to rename_all() and reference the column headers with .. My habit is to keep variable names in lower case and separate words with _.

For string processing, {stringr} will be our workhorse. str_to_lower() will convert our characters to lower-case, and str_replace_all() will take care of of the pesky spaces in our column names.

Here’s reading in the file, renaming all the columns, and assigning the result to a variable that I’m going to call nfl_df:

nfl_df <- read_xlsx(temp_file) %>% 
  rename_all(funs(str_to_lower(str_replace_all(., " ", "_"))))

Phew. After all that, what’s the data actually look like? I’m going to use knitr::kable() and kableExtra::kable_styling() to render nice looking tables, which requires that I use head() to limit output. Otherwise, all the rows will print and you’ll be scrolling forever. That said, you don’t need to use head() inside of R.

  head(10) %>% # keep only the first 10 rows
  kable(format = "html") %>% 
  kable_styling(bootstrap_options = c("striped", "hover", "condensed"),
                font_size = 11,

To make life easier, I’m going to wrap the table formatting into a function that I’ll call prettify().

prettify <- function(df){
  df %>%
    head(10) %>% 
    kable(format = "html") %>% 
    kable_styling(bootstrap_options = c("striped", "hover", "condensed"),
                  font_size = 11,
                  full_width = FALSE)
}

Tidy Up

You’ll notice that the variables (column names) are year and then a bunch of player positions. To get the most out of the data set, we want those positions to be all in one variable, which we’re going to call position.

To do that, we can use tidyr::gather(). gather() takes the arguments key = and value =. Our key = are the player positions and our value = are their salaries. We could name all of those positions explicitly, but there’s an easier way. Since we want to gather all the variables except for year, we can do so by using -year. R will interpret that as “all the columns minus year”.

Here’s what what it looks like in action:

gathered_df <- nfl_df %>% 
  gather(key = position, value = salary, -year)

… and here are our results:
